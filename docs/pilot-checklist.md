# Pilot Checklist

This checklist is for a one-branch pilot.

## Before Pilot

- Create the real legal entity.
- Create the real branch.
- Confirm GSTIN, legal name, registered address, and branch address.
- Configure invoice series for the branch.
- Configure Pool and PS5 resources.
- Configure timed pricing.
- Configure snacks and drinks products.
- Configure GST rates and HSN/SAC codes.
- Create owner, manager, and staff users.
- Confirm staff can open and close shifts.
- Test browser print or receipt printer workflow.
- Test database backup.
- Test database restore into a temporary database.
- Run one full sale: Pool plus PS5 plus snack or drink.
- Run one refund or credit note.
- Run one discount or manager override.
- Close a shift and review totals.
- Open reports and GST/CA exports.

## Pilot Day 1

- Run GameX POS in parallel with the current paper/manual process.
- Compare sales totals, tender totals, GST totals, and shift totals at end of day.
- Record every point where staff hesitates or gets confused.
- Record any missed billing risk when customers add or switch games.
- Record refund, void, discount, and manager override issues.
- Keep paper as the fallback.

## Pilot Day 2-3

- Fix critical workflow issues only.
- Continue daily reconciliation against paper.
- Review operator shift summaries with the manager.
- Review audit logs for unusual actions.
- Decide whether the staff can stop using paper as the primary record.

## After One Week

- Review sales by day and tender type.
- Review GST totals and export format with the CA.
- Review exceptions: voids, refunds, discounts, stock adjustments, and reopened shifts.
- Review Pool and PS5 utilization.
- Review staff feedback.
- Decide whether to roll out to the next branch.
